Implementação: Definir uma Métrica de Desempenho¶

difcil medir a qualidade de um modelo dado sem quantificar seu desempenho durante o treinamento e teste. Isso geralmente feito utilizando algum tipo de mtrica de desempenho, atravs do clculo de algum tipo de erro, qualidade de ajuste, ou qualquer outra medida til. Para este projeto, voc ir calcular o coeficiente de determinao, R2, para quantificar o desempenho do seu modelo. O coeficiente de determinao uma estatstica til no campo de anlise de regresso uma vez que descreve o quo "bom" a capacidade do modelo em fazer estimativas.

Os valores para R2 tm um alcance de 0 a 1, que captura a porcentagem da correlao ao quadrado entre a estimativa e o valor atual da varivel alvo. Um modelo R2 de valor 0 sempre falha ao estimar a varivel alvo, enquanto que um modelo R2 de valor 1, estima perfeitamente a varivel alvo. Qualquer valor entre 0 e 1 indica qual a porcentagem da varivel alvo (ao utilizar o modelo) que pode ser explicada pelos atributos. Um modelo pode dar tambm um R2 negativo, que indica que o modelo no melhor do que aquele que estima ingenuamente a mdia da varivel alvo.

Questão 7 - Busca em Matriz¶

  • O que a tcnica de busca em matriz (grid search)?
  • Como ela pode ser aplicada para otimizar um algoritmo de aprendizagem?

Dica: Quando explicar a tcnica de busca em matriz, tenha certeza que voc explicou o motivo dela ser usada, o que a 'matriz' significa nesse caso e qual o objetivo da tcnica. Para ter uma resposta mais slida, voc pode tambm dar exemplo de um parmetro em um modelo que pode ser otimizado usando essa tcnica.

**Resposta: O K-fold uma tcnica de validao, ou seja, de avaliao do modelo (embora possa ser usado para gerar previses tambm). O conjunto de treino dividido em k subconjuntos e o treinamento do modelo realizado com k-1 desses conjuntos, enquanto a avaliao feita no conjunto restante. Para que seja uma medida robusta, esse processo ser repetido k vezes, usando um dos k subconjuntos diferentes em cada uma das k vezes. Finalmente, ns podemos tirar a mdia da mtrica de avaliao usada para determinar quo bom ficou o modelo. Tambm podemos tirar o desvio padro para saber quo sensvel est o modelo diferentes conjuntos de treinamento.

Em linhas gerais o funcionamento assim:

Randomicamente embaralhe o dataset Divida o dataset em K grupos Para cada grupo: Pegue o grupo como um teste de espera ou teste de dados Pegue os grupos restantes como um conjunto de dados de treinamento Ajuste um modelo no conjunto de treinamento e avalie o conjunto de teste Guarde a pontuao de avaliao e descarte o modelo Resuma a habilidade do modelo usando uma amostra de pontuaes de avaliao do modelo

Os benefcios de usar k-fold cross validation com grid search so:

-A validao por si s, ou seja, avaliar o modelo utilizando um conjunto no visto em treinamento. Se otimizarmos os hiper-parmetros usando uma mtrica de avaliao no conjunto de teste, haveria um "leak" de informao do teste para o treino, o que no bom, pois acaba superestimando a performance do modelo em dados no vistos. Portanto, validar usando k-fold j nos faz evitar esse cenrio; -O k-fold produz uma mtrica mais robusta pelo fato de tirar a mdia de diferentes conjuntos de validao. Como so usados todos os dados de treino como validao em algum momento, temos uma idia melhor da robustez do modelo; - uma boa alternativa para quando no se tem uma grande quantidade de dados e no poderia-se fazer um simples split entre treino, validao e teste com conjuntos suficientemente grandes.

O que poderia dar errado ao utilizar busca em matriz sem um conjunto de validao cruzada a incapacidade de comparao da performance dos modelos gerados com cada conjunto de hiperparmetros, por no haver conjunto de dados de validaco disponvel para uso pela busca em matriz (GridSearch).

Os dados utilizados so divididos em dados de treinamento e dados de teste. O primeiro, contem um resultado (output) conhecido e o modelo aprender baseado nestes dados para poder generalizar e chegar em concluses quando houver um input de novos dados. Para testar o conjunto de dados ns usamos o conjunto de testes, assim podemos validar o quo bem o modelo est aprendendo.

Para garantir que voc est produzindo um modelo otimizado, voc treinar o modelo utilizando busca em matriz para otimizar o parmetro de profundidade mxima ('max_depth') para uma rvore de deciso. Esse parmetro pode ser entendido como o nmero de perguntas que o algoritmo de rvore de deciso pode fazer sobre os dados antes de fazer uma estimativa. rvores de deciso so parte de uma classe de algoritmos chamados algoritmos de aprendizagem supervisionada.

Alm disso, voc ver que a implementao est usando o ShuffleSplit() como alternativa para a validao cruzada (veja a varivel cv_sets). Ela no a tcnica que voc descreveu na Questo 8, mas ela to til quanto. O ShuffleSplit() abaixo ir criar 10 (n_splits) conjuntos misturados e 20% (test_size) dos dados sero utilizados para validao. Enquanto estiver trabalhando na sua implementao, pense nas diferenas e semelhanas com a validao k-fold.

from sklearn.metrics import make_scorer
#from sklearn.grid_search import GridSearchCV #deprecated 
from sklearn.model_selection import GridSearchCV
from sklearn.tree import DecisionTreeRegressor

def fit_model(X, y):
    """ Desempenhar busca em matriz sobre o parmetro the 'max_depth' para uma 
        rvore de deciso de regresso treinada nos dados de entrada [X, y]. """
    
    # Gerar conjuntos de validao-cruzada para o treinamento de dados
    cv_sets = ShuffleSplit(n_splits=10, test_size = 0.20, random_state=0)

    # TODO: Gerar uma rvore de deciso de regresso de objeto
    regressor = DecisionTreeRegressor()

    # TODO: Gerar um dicionrio para o parmetro 'max_depth' com um alcance de 1 a 10
    params = {'max_depth': list(range(1,11))}

    # TODO: Transformar 'performance_metric' em uma funo de pontuao utilizando 'make_scorer' 
    scoring_fnc = make_scorer(performance_metric)

    # TODO: Gerar o objeto de busca em matriz
    grid = GridSearchCV(regressor, param_grid=params, scoring=scoring_fnc, cv=cv_sets)

    # Ajustar o objeto de busca em matriz com os dados para calcular o modelo timo
    grid = grid.fit(X, y)

    # Devolver o modelo timo depois de realizar o ajuste dos dados
    return grid.best_estimator_

Fazendo Estimativas¶

Uma vez que o modelo foi treinado em conjunto de dados atribudo, ele agora pode ser utilizado para fazer estimativas em novos conjuntos de entrada de dados. No caso do regressor da rvore de deciso, o modelo aprendeu quais so as melhores perguntas sobre a entrada de dados, e pode responder com uma estimativa para a varivel alvo. Voc pode utilizar essas estimativas para conseguir informaes sobre os dados dos quais o valor da varivel alvo desconhecida por exemplo, os dados dos quais o modelo no foi treinado.
